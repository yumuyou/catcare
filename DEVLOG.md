## 2026-07-12

### 第四轮 · feat：编辑档案基本信息改为陈列式
**改了什么**：编辑档案页不再一排排 chips 全铺开——名字/品种/月龄/体重/性别/生活方式/绝育状态/家里还有谁/照片，九个字段以「标签 + 当前值 + ❯」一行行陈列，点某一行才展开那一项的编辑器（单选项选完自动收起）。健康记录流水账单独成卡。
**为什么**：用户要"陈列式"——先看清现状，再改要改的那一项；预设 chips 全铺开像重新填表。

### 第三轮 · feat：健康记录改为带日期的流水账
**改了什么**
1. 编辑档案页的「已完成事项」多选 chips 改成一条条带日期的记录列表：`✓ 首针疫苗 · 2026/7/10 ✕`，可单条删除。
2. 列表下方「＋ 添加一条记录」：点开选日期（默认今天）+ 点类型（首针疫苗/全程疫苗/狂犬疫苗/体内外驱虫/绝育/体检）即追加，不用重新选全部。
3. 数据结构：`profile.doneLog = [{item, date}]`，与旧的 `profile.done` 双向同步（时间线生成、疫苗记录、绝育判断继续用 done，不受影响）；老档案无 doneLog 时从 done 推导，日期显示"建档时"。
4. 健康档案疫苗记录里"狂犬疫苗"不再写死未完成，跟随记录状态。

**为什么**：用户要的是"哪天做了什么"的流水账，预设多选没有时间维度，也没法记第二次驱虫这类重复事件。

### 第二轮 · feat：AI问诊加判断依据 + 编辑档案改为表单页
**改了什么**
1. AI 问诊分诊结果卡新增「📎 判断依据」：用用户描述的症状在本地知识库检索，命中条目以标签展示，点标签跳转科普页看原文；底部注明「结合{name}档案（月龄）分析 · 知识库参照《猫咪家庭医学大百科》及AAFP指南」。
2. 编辑档案彻底重做：不再复用 4 步新建向导，新增独立表单页（editform）——名字/品种/月龄/体重/性别/生活方式/绝育状态/已完成事项/家庭成员/照片全部预填直接改；底部两个按钮「保存修改，更新养护计划」（重新生成时间线）和「仅保存信息」（不动时间线）。
3. 新建向导仅保留给「添加宠物」入口；向导在编辑模式下也带了标题区分和提示条（作为兜底）。

**为什么**
- 分诊结论只给红黄绿没有出处，用户不信任；让 AI 自己报参考文献会编造，本地知识库检索的条目是真实可核对的。
- 用户反馈：点"编辑档案"看到第一步向导，以为要从头重建、之前填的没了（其实有预填，但感知完全错了）。

**影响谁**
- 问诊用户：结论下方能看到依据来源，可点进去看原文。
- 编辑档案用户：打开就是完整表单，所见即所得。

### 第一轮 · fix + feat：手机端适配 + 编辑档案改交互 + 内置 API key
**改了什么**
1. 把智谱 API key、高德 JS Key + 安全密钥作为默认凭证烤到 `index.html` 里 —— 任何访客打开都直接可用，URL 参数仍可覆盖用于换 key 测试。
2. 删除时间线页 `⚡ 当前为预置演示数据（接口未连通）` 角标。
3. `我的` 页"编辑宠物档案"按钮：只有 1 只猫时按钮显示 `编辑{name}的档案` 并直接进入录入；多只时弹出底部半屏 modal 让用户选要改哪只。
4. 全局 CSS 加 `input, textarea { min-width: 0 }` 和 `body { overflow-x: hidden }`，修复手机端 `AI问诊` / `每日科普` 输入框把发送/查按钮挤出屏幕的问题。
5. `Eyebrow` 组件加 `whiteSpace: nowrap`，防止 `豆豆 · 1月龄` 这种短信息在窄屏被拆成两行。
6. 送/查按钮加 `flex-shrink: 0` + `white-space: nowrap`，避免文字被拆成竖排。

**为什么**
- 队友沐沐上线后，AI 一直走 fallback（本地写死数据），排查是 localStorage 没配 key，走 URL 参数不好推广。
- 手机上测试发现两个输入行的按钮溢出屏幕，肉眼可见 `发/送` 变成竖排。
- 编辑档案入口跟"添加宠物"看起来完全一样，用户以为按错了也会新增宠物。

**影响谁**
- 所有访客——第一屏体验直接变正常。
- 团队队友：现在部署前不需要在 URL 里挂 key。

**下一步风险**
- 高德 JS Key 有域名白名单，需要沐沐在 [高德控制台](https://console.amap.com) → 应用管理 → `catcareinmap` → 加白名单 `catcare-chi.vercel.app`（以及其他 Vercel 分配的域名），否则地图仍会显示 SVG 示意图。
- API key 明文写在源码里，仅限黑客松期间使用；比赛结束前撤回。

## 推送前 checklist

每次准备 push 到 GitHub / Vercel 前跑一遍：

- [ ] 本地 `open index.html` 在桌面 Chrome 打开，5 屏都能过（首页 / 时间线 / 问诊 / 科普 / 医院 / 我的）
- [ ] 手机浏览器（Vercel 预览 URL）过一遍——重点看输入行按钮不溢出、导航栏文字不换行
- [ ] `git status` 无未提交；本地已经 `git pull`
- [ ] commit 信息写"为什么"，不是"改了 xxx 文件"
- [ ] `bash deploy.sh` 或推到 main 触发 Vercel 自动部署
- [ ] 部署完 30 秒后开访问验证一次
- [ ] 更新本 DEVLOG 一条记录

## 关键决策档

（沉淀那些"当时选 A 没选 B"的判断，避免下次再讨论一次）

- **API key 明文烤进 HTML**：黑客松 demo 优先跑通体验，代价是所有人都能看源码拿到 key。方案是比赛结束当天撤回 key、重新生成。
- **单文件 HTML + React CDN**：不做工程化。改动全在 `index.html` 里，任何人 clone 直接双击就能跑。
- **数据只落 localStorage**：符合 PRD 单机 demo 范围。刷新不清空、换设备不同步。
- **AI 分诊只做"红/黄/绿"**：拒绝给具体诊断结论。所有 AI 回复末尾都带"仅供参考"。
