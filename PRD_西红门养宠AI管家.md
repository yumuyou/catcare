> 版本：v1.0 ｜ 场景：2026西红门镇产业创新黑客松 · 宠物经济赛道 · 从0到1项目
> 用途：① 团队对齐 ② 直接喂给AI辅助开发（本文档所有字段/流程描述均按可实现口径写）
> 验收总原则：**产品能被连贯操作3分钟、录成一条不卡壳的视频**。任何不出现在演示路径上的功能一律降级或砍掉。

## 1. 背景与问题定义

**目标用户**：西红门镇的新手猫主（养猫 < 1年），典型画像：意外开始养猫（捡到/朋友送养），没有系统养猫知识，遇事靠小红书碎片化搜索。

**垂类声明：只做猫。**猫和犬的医疗知识体系差异大——疫苗程序、驱虫方案、急症谱系（如公猫尿闭）、禁忌药物都不同，垂类才做得出可信的分诊。赛道命题一原文即"垂直知识库+多轮对话"，"只做猫"就是对这五个字的直接回答。团队内有真实猫用户做知识校对。

**三个真实痛点**：
1. **不知道该做什么**：疫苗几针、几时驱虫、几月龄换粮——信息散落在搜索结果里，没有一条按自家宠物定制的行动清单
2. **出事了不知道严不严重**：宠物呕吐/软便/精神差，半夜两点该观察还是该冲医院？搜出来的答案从"没事"到"马上送医"都有
3. **该去医院时不知道去哪家**：不熟悉镇内宠物医疗资源，哪家靠谱、哪家24小时、哪家离得近

**解决方案一句话**：录入猫咪档案 → 自动生成专属养护时间线 → 异常时AI问诊分诊 → 需要就医时直接推荐镇内医院。**线上判断，线下落地，闭环在西红门镇内完成。**

**差异化定位**：不是宠物记录工具（记住它），是宠物养护服务（养好它）；且深度绑定西红门镇本地医疗资源，是"镇级民生基础设施"叙事。

## 3. 功能详述

### F1 宠物档案录入
**入口**：首页"开始养宠之旅"按钮。
**表单字段**：
- 生活方式：全室内 / 会出门（单选，图标选择器——出门猫需狂犬疫苗、更高频体外驱虫，**这一题本身就是垂类深度的展示**）
- 品种：下拉+可输入（预置：英短、美短、田园猫、布偶、橘猫、狸花…）
- 昵称：文本
- 月龄：数字（月）
- 体重：数字（kg，一位小数）
- 性别 & 是否绝育：单选
- 照片：上传一张（供档案头像 + F5谷子模块复用）
- 已完成事项（多选）：首针疫苗 / 全程疫苗 / 体内驱虫 / 体外驱虫 / 绝育 / 都没做过

**交互要求**：一屏一步的分步表单（3-4步），每步一个大问题，进度条显示。填完点"生成TA的养护计划"，进入F2，期间展示加载动画（文案示例："正在为豆豆定制养护计划…"）。

### F2 养护时间线生成（核心wow点）
**逻辑**：将档案JSON传入LLM，生成结构化时间线。
**展示**：竖向时间轴，三个分区——
- 🔴 **本周要做**（如：第二针疫苗已到期）
- 🟡 **本月计划**（如：体外驱虫、称重记录）
- ⚪ **未来节点**（如：6月龄绝育评估、换成粮）

每个节点卡片含：事项名 / 建议时间 / 一句话原因 / "设置提醒"开关（仅UI）。

**LLM Prompt要点**（system prompt草稿）：
```
你是一名宠物医疗顾问。根据宠物档案（JSON），生成养护时间线。
规则：
- 严格按品种/月龄/体重/已完成事项计算，不输出与该宠物无关的项
- 输出JSON数组，每项含：{title, period(week/month/future), suggested_date, reason, category(疫苗/驱虫/喂养/体检/其他)}
- reason控制在20字内，口吻温和不吓人
- 只输出JSON，不要任何其他文字
```
**验收**：输入"2月龄英短、未打疫苗"，能输出含疫苗程序（3针间隔）、驱虫、幼猫粮喂养的合理时间线；换成"3岁金毛、疫苗齐全"输出明显不同。

### F3 AI问诊 + 分诊
**入口**：时间线页底部常驻按钮"TA好像不太对劲？"＋问诊tab。
**交互**：对话界面，预置3个快捷症状chip（呕吐了 / 不吃东西 / 尿不出来）+ 自由输入。AI追问1-2轮（吐了几次？精神状态？）后给出**分诊卡片**：

- 分诊等级三档：🟢 在家观察（附观察要点）/ 🟡 24-48h内就医 / 🔴 立即就医
- 🟡/🔴 等级下方直接出现"查看附近医院"按钮 → 跳转F4

**Prompt要点**：结合宠物档案上下文（月龄很关键：幼猫呕吐比成猫危险）；固定输出格式{追问 or 分诊结论}；**结论必须带免责声明一行**："AI分诊仅供参考，紧急情况请直接就医"。
**验收**："2月龄猫吐了3次没精神" → 必须给🔴；"公猫一天没排尿、频繁蹲砂盆" → 必须给🔴（猫科经典急症，路演可当王牌案例）；"成年猫偶尔软便精神好" → 🟢。

### F4 医院推荐页（闭环终点）
**数据**：硬编码JSON，字段：`{name, address, phone, is24h, lat, lng, distance_text}`。镇内起步名单（录入前高德核实营业状态）：
- 全心全意连锁动物医院·24小时西红门店（宏业路理想家园北门底商）★24h
- 全心全意动物医院总院（欣宁街72号）
- 宠颐生爱福动物医院（宏业路瑞海家园三区北门底商）
- 康尔贝动物医院（欣荣北大街18-12号）
- 西红门百灵动物医院（宏旭路248号）
- 西红门新星动物医院（西红门镇五村）

**展示**：上半屏简版地图（高德JS API嵌入；降级方案：静态地图图+CSS标记；再降级：纯卡片列表），下半屏医院卡片按"24小时优先+距离"排序。🔴分诊跳转过来时，24小时医院置顶并高亮"现在营业"。卡片含一键拨号按钮（tel: 链接）。

### F7 就诊回填（闭环回流）
**入口**：医院卡片上"记录本次就诊"按钮（就诊后使用）。
**表单**：就诊医院（自动带入）/ 日期（默认今天）/ 诊断（一句话）/ 医嘱（自由文本）。
**核心逻辑**：医嘱文本传入LLM解析为1-4条结构化待办（如"3天后复诊"→本周节点、"益生菌连喂5天"→本周节点），**自动插入F2时间线顶部并打🏥医嘱标**；就诊记录沉淀进F6健康档案。
**Prompt要点**：输出JSON数组 {title, period, suggested_date, reason, category:"医嘱"}；解析失败兜底为一条"按医嘱复诊"。
**叙事价值**：这是"闭环"的下半段——线上判断→线下就医→**结果回流→时间线更新**。路演时一句话讲清："医生说的话，不会留在诊室里，会变成第二天早上的提醒。"
**验收**：输入"禁食12小时，益生菌每天两次连喂5天，3天后复诊"，时间线本周区出现复诊和喂药待办。

### F5 谷子预览（P1彩蛋）
档案照片 → 生成吧唧/冰箱贴预览图（圆形裁切+边框模板即可，**优先用canvas贴模板实现，不赌生图API**）。入口放时间线页底部"把TA带在身边"。商业化叙事：1688供应链定制 + 本地宠物文创经济。
**死线**：周六15:00未跑通 → 移入deck商业化页，不进产品。

### F6 健康档案页（P2静态）
展示：体重曲线（写死3个数据点的折线图）、疫苗记录表、问诊历史列表。全静态，路演口径："问诊和时间线的每条记录都会沉淀在这里"。

## 4. Demo演示路径（=录屏脚本，3分钟）

1. 首页 → 点"开始养宠之旅"（15s）
2. 分步填写档案：2月龄英短"豆豆"，用沐沐家猫真实照片（40s）
3. 时间线生成：加载动画 → 时间轴展开，指出"第二针疫苗本周到期"（40s）
4. 点"TA好像不太对劲" → 选"呕吐了" → AI追问 → 🔴立即就医（45s）
5. 跳医院页：地图+24小时店置顶+"距您1.2km 现在营业"+拨号按钮（30s）
6. （若F5存活）点"把TA带在身边" → 吧唧预览一闪而过（10s）

**录制要求**：≥1080p，全程无卡顿无报错弹窗；LLM响应用预缓存结果兜底，不现场赌接口。

## 5. 技术方案

- **栈**：Next.js + Tailwind（单页应用，路由：/ → /onboarding → /timeline → /consult → /hospitals → /archive）
- **LLM**：现有可用API（组委会发的Opendev额度），前端直调或Next.js API route转发；**所有LLM输出要求纯JSON并做try-catch兜底**——解析失败时展示预置的缓存结果，绝不白屏
- **状态**：宠物档案存内存state + localStorage（仅本地运行；如部署为线上demo则只用内存state）
- **地图**：高德JS API（需申请key，今晚申请）；降级路径见F4
- **部署**：Vercel或本地跑，以录屏顺畅为准
